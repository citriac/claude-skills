## 国内平台详细指南

### 1. 掘金 (juejin.cn)

#### 用户特征（2026）
- **年龄**: 25-35岁为主
- **职业**: 前端、后端、移动端开发者
- **关注**: 新技术、实战经验、职业发展
- **活跃时间**: 工作日晚上，周末全天

#### 内容策略
- **最佳内容类型**:
  1. 技术实战教程（步骤清晰，代码完整）
  2. 开源项目展示（GitHub链接+使用说明）
  3. 性能优化经验（实际案例+数据对比）
  4. 开发工具推荐（对比评测+使用心得）

- **标题技巧**:
  - 包含技术栈关键词：如"Vue 3.x"、"Spring Boot"
  - 突出价值：如"提升50%性能"、"减少80%代码"
  - 使用数字：如"10个实用技巧"、"5分钟学会"

- **标签系统**:
  - 最多5个标签
  - 必须包含技术栈标签
  - 可选：前端、后端、算法、数据库等分类标签

#### API限制
- **认证方式**: OAuth 2.0 + JWT Token
- **发布频率**: 100次/天（个人账号）
- **内容审核**: 自动审核，敏感词过滤
- **图片限制**: 单图≤10MB，支持JPG/PNG/WebP

#### 最佳实践
1. **代码格式化**: 使用掘金内置代码高亮
2. **目录结构**: 长文章建议添加目录
3. **图片优化**: 压缩图片，添加alt文本
4. **互动引导**: 结尾添加问题或讨论话题

### 2. 知乎 (zhihu.com)

#### 用户特征（2026）
- **学历**: 本科以上占比85%
- **行业**: 互联网、金融、教育、科技
- **偏好**: 深度思考、逻辑分析、数据支持
- **阅读习惯**: 长文阅读，关注评论区讨论

#### 内容策略
- **技术专栏建设**:
  1. 专栏主题明确（如"AI编程实践"）
  2. 系列文章保持一致性
  3. 定期更新（建议每周1-2篇）
  4. 专栏简介清晰，突出价值

- **回答 vs 文章**:
  - **回答**: 针对具体问题，直接解决方案
  - **文章**: 系统性论述，完整知识体系

- **格式要求**:
  - 标题层级清晰（H1-H4）
  - 代码块使用正确语法标注
  - 数据可视化（图表、流程图）
  - 参考文献规范引用

#### API限制
- **认证**: OAuth 2.0，需要申请开发者权限
- **频率**: 50次/天（企业级API更高）
- **内容长度**: 文章≤5万字，回答≤2万字
- **媒体**: 支持Markdown，图片需知乎图床

#### 合规要求
1. **专业认证**: 技术领域建议完成专业认证
2. **引用规范**: 注明数据来源，尊重版权
3. **风险提示**: 涉及安全、性能等需添加提示
4. **更新维护**: 技术过时内容需标注或更新

### 3. CSDN (csdn.net)

#### 用户特征（2026）
- **新手友好**: 大量学生和初学者
- **问题导向**: 解决具体开发问题
- **搜索引擎**: 百度SEO权重高

#### 内容策略
- **入门教程**: 步骤详细，从零开始
- **问题解决**: 常见错误解决方案
- **配置指南**: 环境搭建、工具配置
- **代码片段**: 可直接使用的代码

#### SEO优化
- **关键词**: 在标题、首段、小标题中嵌入
- **结构**: H1-H6层级清晰，目录自动生成
- **内外链**: 内部相关文章链接，外部权威引用
- **元数据**: 摘要简洁，包含主要关键词

## 国际平台详细指南

### 1. Reddit (reddit.com)

#### 子版块策略
- **r/programming**: 通用编程话题
  - 规则：禁止求职、课程推广
  - 内容：开源项目、技术新闻、深度文章
  - 发布时间：UTC 14:00-18:00

- **r/webdev**: 网站开发
  - 规则：展示项目需提供源码
  - 内容：前端框架、性能优化、工具推荐
  - 标签：[Showcase]、[Tutorial]、[Discussion]

- **r/Python**: Python社区
  - 规则：代码分享需完整可运行
  - 内容：库推荐、最佳实践、学习资源
  - 高质量讨论：解释原理，提供基准测试

#### API使用规范（2026）
- **认证**: OAuth 2.0，需要User-Agent头
- **频率限制**: 100次/分钟（认证后）
- **用户代理**: 必须包含开发者信息
- **遵守规则**: Reddit API使用条款

#### 成功要素
1. **参与讨论**: 不只是发布，要参与回复
2. **避免spam**: 同一内容不在多个子版块重复发布
3. **透明身份**: 如使用自动化工具，在帖子中说明
4. **尊重社区**: 遵守各子版块具体规则

### 2. Dev.to (dev.to)

#### 社区文化
- **开源友好**: 鼓励分享完整代码
- **学习氛围**: 允许"学习笔记"式文章
- **互动积极**: 评论质量高，建设性反馈

#### 内容格式
- **标签系统**: 完善的标签体系，支持#beginners #tutorial #showcase
- **系列文章**: 支持文章系列功能
- **代码嵌入**: 支持GitHub Gist、CodePen、Replit
- **封面图片**: 建议添加相关封面图

#### 发布技巧
1. **标题**: 清晰描述内容，避免clickbait
2. **摘要**: 前200字决定是否继续阅读
3. **结构**: 使用目录，长文分章节
4. **结尾**: 添加讨论问题或行动号召

### 3. Hacker News (news.ycombinator.com)

#### 精英社区特征
- **高质量标准**: 对内容质量要求极高
- **技术深度**: 偏好技术创新和深度分析
- **创业导向**: 关注技术商业化、产品化

#### 提交策略
- **标题**: 简洁客观，避免主观形容词
- **内容**: 技术细节充分，数据支撑有力
- **时机**: 美西时间上午，避开周末
- **互动**: 准备回答深入技术问题

#### 注意事项
1. **避免自我推广**: 以分享知识为主，而非推广
2. **原创优先**: 原创内容比转载更受欢迎
3. **技术焦点**: 保持技术讨论，避免偏离
4. **尊重规则**: 遵守HN社区指南

## 通用最佳实践

### 内容质量
1. **准确性**: 技术细节必须准确
2. **实用性**: 提供可操作的解决方案
3. **完整性**: 文章结构完整，逻辑清晰
4. **时效性**: 技术内容保持更新

### 发布管理
1. **时间规划**: 根据目标受众时区安排
2. **平台差异化**: 同一内容不同平台不同角度
3. **频率控制**: 避免过度发布导致疲劳
4. **效果追踪**: 使用UTM参数跟踪来源

### 合规要求
1. **版权尊重**: 使用图片、代码需遵守许可
2. **隐私保护**: 不包含用户敏感信息
3. **平台规则**: 定期检查各平台规则更新
4. **法律合规**: 遵守当地法律法规

## 监控与优化

### 关键指标
1. **浏览量**: 基础曝光指标
2. **互动率**: 点赞/评论/分享比例
3. **转化率**: 引导到目标网站的比例
4. **留存率**: 用户返回阅读的比例

### 优化循环
```python
# 简化的优化流程
1. 发布内容
2. 收集数据（24小时）
3. 分析表现（对比基准）
4. 调整策略（标题/内容/时间）
5. A/B测试验证效果
6. 更新最佳实践
```

1. **Google Analytics**: 流量分析
2. **Ahrefs/SEMrush**: SEO分析
3. **Buffer/Hootsuite**: 社交媒体管理
4. **自定义脚本**: 平台特定数据分析
